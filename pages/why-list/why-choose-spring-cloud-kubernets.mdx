사실 Discovery Server, Config Server 와 같은 개념은 Kubernetes 레벨에서 제공해줍니다. 그런데 이렇게 Kubernetes 레벨에서 적용을 하면 애플리케이션이 인프라에 종속적으로 되어서 기능 수정을 해야 하거나 애플리케이션의 특정 Property 들을 수정해야 할 때 인프라 (K8S 리소스) 를 수정해야 하는 상황이 발생합니다.<br/> 

이런 이유로 인해 Pivotal 에서는 Spring Cloud Kubernetes 를 출시했습니다.<br/>
Discovery Server, Config Server 를 Spring Cloud Kubernetes 에서 애플리케이션 레벨에서 구현한다면, 인프라(EKS) 레벨과 애플리케이션(Spring) 레벨의 변경사항이 스파게티 처럼 얽히는 일은 줄어들 것 입니다.<br/>
