앱의 골격은 Flutter 를 기반으로 하지만, 내부의 웹뷰는 React 를 사용하기로 했습니다.

웹뷰를 사용하기로 한 이유는 아래와 같습니다.
- 1인 개발 체제 특성상 시간을 아껴야 하기에 앱 재배포 대응에 오랜 시간이 걸리지 않도록 하기 위해서
<br/>

React 를 웹뷰에서 사용하기로 한 이유는 아래와 같습니다.
- 시간 날때마다 조금씩 스터디를 했던 프레임워크여서
<br/>

그리고 React 를 Serving 해줄 Node.js 계열의 서버프레임워크도 필요한데, Next.js 를 선택했습니다. Next.js 를 선택한 이유는 별다른 이유가 없습니다. 단지 아래의 이유 때문입니다.
- Node.js / Express.js 로 모든걸 짜기에는 시간이 아깝고, Next.js 는 그나마 예전에 공부해본 적이 있어서
