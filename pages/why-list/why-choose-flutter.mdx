처음에는 React 를 생각했었지만, React Native 를 배우는 것과 CSS 의 어려움 때문에 일단 멈췄습니다.<br/>
아직까지는 백엔드에 치중된 경력이기에 CSS 의 경우 유지보수는 실무에서 많이 해왔었지만, 실제 스타일시트를 유지보수가 편하도록 구성하는 것은 실무에서 해보진 않았기 때문입니다.<br/>

반면 Flutter 를 선택하게 된 이유는 이렇습니다.
- 디자인에 대해서 Material Design 이라는 하나의 템플릿이 제공된다. 무료로 사용가능하다.
<br/>

단지 이 이유 하나 때문입니다.<br/>
디자인 공수를 최대한 줄여야 했기 때문에 dart 언어를 새로 배우는 어려움을 감수하더라도 올해는 Flutter 를 선택했습니다.<br/>
배포도 새로 배워야 하고 Flutter 프레임워크를 학습하고, 테스트 방식들을 학습해야 하고, 레이아웃도 배워야 하는 그런 단점들이 있지만, 이 것은 나중에 1인 스타트업을 하게 될 수 있는 상황도 생각한다면 하나의 과정에 지나지 않는다는 생각이 들어서 일단은 Flutter 를 선택하게 되었습니다.<br/>

다만 CSS 를 새로 만든다는 것은 저에게 너무나 큰 부담이었...습니다. 색상에 대한 센스 자체도 부족하고, 목업을 어떻게 잡을지에 대한 센스나 경험이 부족하기 때문입니다.<br/>
<br/>

그런데 아직도 저는 React 에 관심이 많기 때문에 어떻게든 실무에서 접점을 찾을 것 같고 내년에는 또 새로운 앱을 만들어보려고 하는데 그 때는 React Native 를 선택해보게 되지 않을까 싶습니다. 그 때까지는 CSS 를 처음부터 다시 공부해보면서 잘알려진 개념들을 정리해두고 싶은 욕심도 꽤 많기에 React Native 기반의 새로운 앱을 만들어보는 것은 내년으로 미루기로 했습니다.<br/>
